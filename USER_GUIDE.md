# Audio Spectrum Overlay Maker v1.3.0 User Guide

Audio Spectrum Overlay Maker は、音源ファイルを解析し、音に反応して動くスペクトラムアナライザー風の **無音 MP4 素材** を生成するローカルアプリです。

このアプリの目的は、完成動画を直接作ることではありません。Suno などで作成した音楽、オリジナル曲、アルバム動画、歌詞動画、配信用動画などに重ねるための、動画編集ソフト向けスペアナ素材を作ることです。

生成される MP4 には音声は入りません。音源は解析にだけ使われます。最終的な合成、背景画像、字幕、音声トラック、書き出しは、PowerDirector / VideoStudio / DaVinci Resolve などの動画編集ソフト、または同梱の SRT Spectrum Video Composer 側で行います。

## このアプリでできること

Audio Spectrum Overlay Maker では、次のことができます。

- 音源ファイルを読み込み、音量・周波数成分に反応するスペアナ動画を生成する。
- 黒背景にスペアナバーを描いた、動画編集ソフト向けの無音 MP4 を生成する。
- 比較合成用に、メイン動画とマット動画のペアを生成する。
- 片側バー / 両側バーを選択する。
- 縦グラデーション / 帯域グラデーション / ループ帯域グラデーションを使う。
- 周波数帯域を左右にスクロールさせる。
- LED / デジタル表示風に、バーをセグメント表示する。
- ピークホールドで、直近ピーク位置に短いピーク片を残す。
- Post Transform layer で、描画済みフレーム全体に回転・縦台形・横台形・音量連動拡大を適用する。
- 回転と台形変換を組み合わせた、斜め投影風のスペアナ素材を作る。
- 音量連動の拡大で、キックやベースに合わせた拍動表現を作る。
- 見た目プレビューで静止状態を確認する。
- 実際の音源を使った2秒 / 10秒の動きプレビューを GUI 内で再生する。
- 30秒プレビュー MP4 を生成して、動画編集ソフトで重なり方を試す。
- 全体生成で、音源全体に同期したスペアナ素材を作る。
- ペア出力時にメイン動画とマット動画を並列生成する。
- システムプリセットとユーザープリセットを使う。
- 同梱の SRT Spectrum Video Composer をボタンから開く。
- 日本語 / English の UI を切り替える。

## 出力される素材

このアプリが出力する基本素材は、**音声なし MP4** です。

音源ファイルは、スペアナの動きを作るための解析にだけ使います。生成された動画には音声が含まれないため、動画編集ソフトでは、元の音源を別トラックに配置してください。

通常出力は次のような素材です。

```text
黒背景 + スペアナ本体
```

動画編集ソフト側で「比較（明）」「Lighten」「Screen」などの合成を使うと、黒背景を実質的に消し、スペアナ部分だけを背景画像や動画の上に重ねられます。

ただし、背景動画が明るい場合や、色付きバーを使う場合、通常の比較（明）だけではスペアナが背景に負けることがあります。そのため、現在の標準運用では、次の **比較合成用ペア出力** を推奨します。

```text
メイン動画: 黒背景 + 通常のスペアナ
マット動画: 白背景 + 黒いスペアナ形状
```

## 基本ワークフロー

まずは次の流れで使うのがおすすめです。

1. `start.bat` でアプリを起動する。
2. 「音源」で解析する音声ファイルを選ぶ。
3. 「出力先」で保存先フォルダを選ぶ。
4. プリセットを選ぶ。最初は `01 Basic White` が安全です。
5. 「見た目プレビュー」で形と色を確認する。
6. 重い設定を試す場合は「2秒プレビュー」、通常確認では「10秒プレビュー」で実際の音源に対する動きを確認する。
7. 「30秒プレビュー動画生成」で短い MP4 を作る。
8. 動画編集ソフトで重ね方を確認する。
9. 問題なければ「全体生成」を行う。
10. 必要なら「SRT Spectrum Video Composer を開く」で合成用アプリへ渡す。

現在は「比較合成用マットも出力」が標準で ON になっています。通常はこのまま使うことを推奨します。

## インストールと起動

### 1. ZIP を展開する

配布 ZIP を任意のフォルダに展開します。

フォルダ名に日本語が含まれていても基本的には動作しますが、トラブルを避けたい場合は、英数字中心のパスに置くと安全です。

### 2. 初回セットアップ

Windows では、同梱の `setup.bat` を実行します。

`setup.bat` は次を行います。

- `py -3` が使えるか確認する。
- Python 3.10 以上か確認する。
- `.venv` がなければ作成する。
- `pip` を更新する。
- `requirements.txt` の依存パッケージをインストールする。

Python は `py -3` で起動されます。Python 2.7 や意図しない Python が選ばれないようにするためです。

確認例:

```bat
py -3 --version
```

Python 3.10 以上が表示されることを確認してください。

### 3. 起動

通常は `start.bat` を実行します。

`start.bat` は、`.venv` がない場合に `setup.bat` を自動実行し、その後 `.venv` 内の Python で `app.py` を起動します。

手動で起動する場合:

```bat
py -3 app.py
```

## ffmpeg / ffprobe の参照順

アプリは `ffmpeg` と `ffprobe` を次の順で探します。

1. アプリ所在地の `bin` フォルダ
   - `bin\ffmpeg.exe`
   - `bin\ffprobe.exe`
2. PATH

EXE 化した場合も同じ考え方です。EXE と同じ場所に `bin` フォルダを作り、その中に `ffmpeg.exe` と `ffprobe.exe` を置くと、PATH を変更せずに使えます。

PyInstaller の `--windowed` EXE でも、ffmpeg / ffprobe 起動時に黒いコマンドプロンプト窓が出ないようにしています。

## 音源と出力先

### 音源

「音源」には、解析する音声ファイルを指定します。

対応形式は、ローカルの ffmpeg が読み込める形式です。例:

- WAV
- MP3
- M4A
- AAC
- FLAC
- MP4 内の音声

実運用では、WAV / MP3 / M4A が扱いやすいです。

### 出力先

「出力先」には、生成した MP4 を保存するフォルダを指定します。

未指定の場合は、音源ファイルと同じフォルダに保存されます。

「出力先を開く」ボタンで、現在の出力先フォルダを開けます。

## プリセット

プリセットは、見た目・動き・詳細設定をまとめて呼び出す機能です。

### システムプリセット

現在のシステムプリセットは次の通りです。

- `01 Basic White`
- `02 Dual White`
- `03 Blue Gradient`
- `04 Neon Dual`
- `05 Scrolling Spectrum`
- `06 Smooth Ambient`
- `07 Digital LED`
- `08 Digital Dual`
- `09 Digital Neon Scroll`
- `10 Trapezoid Neon`
- `11 Vertical Scroll`
- `12 Pulsating LED Scroll`

システムプリセットは削除できません。

### プリセットの目安

`01 Basic White`  
最も安全な基本形です。白バー、黒背景、片側バーです。まずはこれを基準にしてください。

`02 Dual White`  
中央線から上下に伸びる両側バーです。左右ではなく上下対称の存在感を出したい場合に向きます。

`03 Blue Gradient`  
青系の縦グラデーションです。落ち着いた音楽動画に使いやすい設定です。

`04 Neon Dual`  
派手な両側ネオン系です。ピークホールドも有効です。

`05 Scrolling Spectrum`  
帯域が横方向にスクロールする設定です。ループ帯域グラデーションにより、左右端の色がつながりやすくなっています。ピークホールドも有効です。

`06 Smooth Ambient`  
落ち着いた動きです。長尺BGM、Lo-fi、バラードなどに向きます。

`07 Digital LED`  
LEDメーター風のセグメント表示です。ピークホールドも有効です。

`08 Digital Dual`  
両側バー + デジタル表示です。ピークホールドも有効です。

`09 Digital Neon Scroll`  
デジタル表示、ネオン配色、スクロールを組み合わせた派手な設定です。ピークホールドも有効です。

`10 Trapezoid Neon`  
デジタル表示、ネオン配色、縦台形変換を組み合わせたプリセットです。Post Transform の台形変換を確認する基準にも使えます。

`11 Vertical Scroll`  
回転によって縦方向のスペアナに見せるプリセットです。両側バー、スクロール、ループ帯域グラデーションを組み合わせています。

`12 Pulsating LED Scroll`  
デジタル表示、スクロール、音量連動の拡大を組み合わせたプリセットです。低域側の音量を参照し、キックやベースに合わせた拍動を狙う設定です。

### ユーザープリセット

現在の設定を保存するには、「プリセット保存」を押します。

ユーザープリセットは削除できます。システムプリセットは削除できません。

プリセットを選んだあとに設定を変更すると、プリセット欄は `カスタム` になります。これは、現在の設定が選択中のプリセットと一致しなくなったことを示します。

### プリセットに保存されるもの

保存される代表例:

- 横幅 / 高さ / fps
- バー本数
- 表示タイプ
- 色モード
- バー色
- バー幅
- 角丸
- デジタル化
- ピークホールド
- 回転角度 / 縦台形 / 横台形
- 音量連動の拡大
- 音量拡大率 / 閾値 / 天井 / 保持 / 減衰
- 低域のみ参照
- 最大高さ / 余白
- 反応の速さ
- 感度
- 詳細設定
- エンコード設定

保存されない代表例:

- 音源ファイル
- 出力先フォルダ
- 言語
- 動きプレビューの開始位置
- プレビュー秒数
- ログ

## プレビュー

### 見た目プレビュー

左上の静止プレビューです。

現在の見た目設定を使い、スペアナ素材の静止状態を表示します。

確認できるもの:

- 画面サイズ
- バー本数
- 片側 / 両側
- バー幅
- 色
- 角丸
- デジタル表示
- ピークホールドの見た目
- 余白
- 最大高さ

### 2秒プレビュー / 10秒プレビュー

左下のプレビューです。

実際の音源を短く解析し、スペアナの動きを GUI 内で一度だけ再生します。

v1.3.0 では、ボタンが2つあります。

- `2秒プレビュー`: 回転、台形変換、音量連動拡大など、重い設定を素早く確認するための短いプレビューです。
- `10秒プレビュー`: 通常確認用の動きプレビューです。

「動きプレビューに適した区間を自動検出」が ON の場合、音源の先頭付近をスキャンし、動きが確認しやすい区間を使います。静かなイントロが長い曲でも確認しやすくなります。

このプレビューは確認用です。本番の MP4 は「30秒プレビュー動画生成」または「全体生成」で作ります。

### プレビュー基準

上部の「プレビュー基準」では、左ペインのプレビューをどの基準で表示するか選べます。

- 高さで揃える
- 幅で揃える

横長素材や縦長素材を確認するときに便利です。

## 比較合成用ペア出力

このアプリの重要機能です。

「比較合成用マットも出力」を ON にすると、通常のメイン動画に加えて、マット動画も同時に出力します。現在は標準で ON です。

出力されるペア:

```text
メイン動画:
  黒背景
  通常のスペアナ
  バー色・グラデーション・デジタル表示・ピークホールドを反映

マット動画:
  白背景
  スペアナとして表示したい形状だけ黒
  メイン動画と同じバー形状・同じピークホールド形状
```

動画編集ソフト側では、次の順に重ねます。

```text
下: 完成済みの背景動画
中: マット動画  → 比較（暗） / Darken
上: メイン動画  → 比較（明） / Lighten
```

この方式では、まずマット動画でスペアナの形状部分だけを暗く抜き、その上にメイン動画を乗せます。明るい背景でもスペアナが負けにくく、クロマキーよりもエッジが安定しやすい方式です。

重要な仕様:

- ペア出力 ON/OFF にかかわらず、メイン動画は通常のスペアナ素材です。
- ペア出力 ON 時は、メイン動画に加えてマット動画が追加されます。
- メインとマットは同じ解析結果を使います。
- ピークホールドも、メインとマットで同じ形状になります。
- 出力動画に音声は入りません。

## SRT Spectrum Video Composer 連携

同梱の `final_composer.py` は、SRT Spectrum Video Composer です。

これは、次の素材をまとめて最終 MP4 を作るための別アプリです。

- 背景画像
- 音源
- SRT字幕
- カラースペアナ動画
- マット動画

Audio Spectrum Overlay Maker からは、下部の「SRT Spectrum Video Composer を開く」ボタンで起動できます。

### 連携の考え方

Audio Spectrum Overlay Maker は、SRT Spectrum Video Composer に次の3つだけを渡します。

- 音源ファイル
- 直近に生成したメインスペアナ動画
- 直近に生成したマット動画

背景画像、SRT字幕、出力MP4、レイアウトなどは Composer 側で選びます。

まだ音源やスペアナ動画がない状態でもボタンは押せます。その場合、handoff JSON の該当項目は空欄で渡されます。

### 内部的な起動方法

ボタンを押すと、アプリは一時 handoff JSON を次の場所に作ります。

```text
%LOCALAPPDATA%\AudioSpectrumOverlayMaker\handoff\
```

その後、このアプリ自身を次の引数付きで別プロセス起動します。

```text
--srt-spectrum-video-composer --handoff <json>
```

通常起動時には `final_composer.py` は import されません。引数付きで起動された子プロセスだけが `final_composer.py` を読み込みます。

これにより、Audio Spectrum Overlay Maker と SRT Spectrum Video Composer の相互依存を小さくしています。

PyInstaller で EXE 化した場合は、次の順に `final_composer.py` を探します。

1. PyInstaller の展開先フォルダ
2. EXE と同じフォルダ
3. `app.py` と同じフォルダ

`--add-data` で `final_composer.py` を同梱した場合も、EXE横に外部ファイルとして置いた場合も起動できるようにしています。

SRT Spectrum Video Composer の詳しい使い方は、同梱の `USER_MANUAL_SSVC.md` を参照してください。

## 見た目タブ

見た目タブでは、生成されるスペアナ素材の外観を調整します。

### 表示タイプ

- 片側バー
- 両側バー

片側バーは、下の基準線から上に伸びる通常の表示です。

両側バーは、中央線から上方向と下方向に同じバーが伸びます。ピークホールドも上下両方に出ます。

### 色モード

色の付け方を選びます。

#### 縦グラデーション

バーの縦方向に色を変えます。

`バー色1` と `バー色2` が同じ場合は単色になります。

#### 帯域グラデーション

横方向に、左端 `バー色1`、右端 `バー色2` で線形補間します。

スクロールしない帯域表示や、左右方向に色変化を見せたい場合に使います。

#### ループ帯域グラデーション

横方向の両端が `バー色1`、中央が `バー色2` になるように補間します。

色配置は、おおまかに次のようになります。

```text
左端: 色1
中央: 色2
右端: 色1
```

スクロール時に左端と右端がつながっても、色が不連続になりにくいことを目的にしています。

### スクロール

バーの位置は固定したまま、担当する周波数帯域を左右にローテーションします。

- なし
- 左
- 右

「シフト間隔」で、何フレームごとに帯域を1つ動かすかを指定します。

### バー本数

表示するバーの本数です。

選択肢:

- 18
- 24
- 32
- 48
- 64

少ない本数ほど太くシンプルに見えます。多い本数ほど細かく、密度の高いスペアナになります。

デジタル系プリセットでは、LEDメーターらしさを出すために32本を使っています。

### バー色1 / バー色2

バーの色を指定します。

白バーは最も合成しやすく、基本用途に向きます。色付きバーは見栄えがよい反面、動画編集ソフトの合成モードによって見え方が変わる場合があります。

迷ったら、まず白→白で合成確認し、その後に色を付けるのが安全です。

### バー幅%

各バーの太さです。

値を下げると、バーが細くなり、間隔が広がります。値を上げると、バーが太くなり、密度が高く見えます。

### 角丸半径px

バーの角丸半径です。

通常バーでは、値を大きくするとカプセル形状に近づきます。低信号時には、いきなりフル幅の四角として出ず、中心から小さなピル形状として現れます。

デジタル表示では、角丸は通常 0 の方がLEDメーターらしくなります。

### デジタル化

バーを縦方向のセグメントに分割し、LED / デジタルメーター風に表示します。

通常バーでは、バーの途中まで連続的に塗られます。デジタル化をONにすると、各セグメントは ON / OFF のどちらかだけになります。

### 分割数

1本のバーを縦方向に何分割するかを指定します。

値を増やすほど細かいLED表示になります。

### ギャップpx

セグメント同士の間に入れる隙間です。

0にすると隙間なしの段階表示になります。2px前後にするとLEDメーターらしくなります。

### 回転角度°

描画済みの1フレーム全体を、キャンバス中心のまわりに回転します。

`0` の場合は回転しません。右側の「リセット」ボタンで `0` に戻せます。

キャンバスサイズは変わりません。はみ出した部分はクリップされ、空いた領域はメイン動画では黒、マット動画では白で埋められます。

### 縦台形

上下方向の台形変換です。

- `-100` に近いほど上部が小さくなります。
- `0` で変形なしです。
- `+100` に近いほど下部が小さくなります。

右側の「リセット」ボタンで `0` に戻せます。

### 横台形

左右方向の台形変換です。

- `-100` に近いほど左側が小さくなります。
- `0` で変形なしです。
- `+100` に近いほど右側が小さくなります。

右側の「リセット」ボタンで `0` に戻せます。

回転、縦台形、横台形は同時に使えます。処理順としては、長方形のスペアナ素材を台形に写像し、その形を保ったまま回転するイメージです。

### 音量連動の拡大

ONにすると、音量に応じて Post Transform の拡大率を変化させます。デフォルトはOFFです。

この機能は、バーの高さとは別に、スペアナ素材全体を少し大きくしたり小さくしたりする演出です。

### 音量拡大率%

音量連動時に到達する倍率です。

- `100%`: 変化なし
- `101%` 以上: 音が大きいほど拡大
- `99%` 以下: 音が大きいほど縮小

設定範囲は `80%` から `150%` です。標準は `115%` です。

### 拡大開始閾値

音量連動の変化が始まる基準です。この値以下の音量は、拡大率計算ではゼロとして扱います。

標準は `10%` です。小さくすると反応しやすくなり、大きくすると強い音だけに反応します。

### 最大拡大天井

この値以上の音量を、最大拡大率として扱います。

標準は `50%` です。天井を下げると最大倍率に到達しやすくなります。天井を上げると、本当に大きい音だけが最大倍率に届きます。

### 音量保持ms / 音量減衰ms

音量連動拡大用のピークホールド的なパラメータです。

`音量保持ms` は、大きい音が入ったあと、その拡大状態をどの程度保持するかを指定します。標準は `66ms` です。

`音量減衰ms` は、保持後に元の大きさへ戻る速さです。標準は `167ms` です。

通常のピークホールドよりも機敏に動く値を使うと、音に合わせた拍動として見えやすくなります。

### 低域のみ参照

ONにすると、音量連動拡大の判定に低域側のバーだけを使います。デフォルトはONです。

キックやベースに合わせてスペアナ全体を脈動させたい場合はONがおすすめです。参照する低域範囲は、詳細設定タブの「低域参照範囲%」で調整します。

### 最大高さ%

バーが使用する最大高さです。

値を大きくすると、ピーク時のバーが高くなります。値を小さくすると、全体を控えめにできます。

### 左右余白%

スペアナ全体の左右余白です。

### 下余白%

片側バーの基準線から下端までの余白です。

両側バーでは、上下対称表示を優先するため、外側余白として扱われます。

### 横幅 / 高さ / fps

生成する動画素材のサイズとフレームレートです。

通常は 30fps で十分です。60fps にすると動きは滑らかになりますが、生成時間とファイルサイズが増えます。

## ピークホールド

ピークホールドは、各バーの直近ピーク位置に短い表示を残す機能です。

バー本体の動きを変えるのではなく、バー本体とは別に「ピーク片」を描きます。

基本的な動き:

```text
1. 現在のバーがピーク値以上になった
   → ピーク位置を更新
   → 保持時間をリセット

2. 現在のバーがピークより低く、保持時間内
   → ピーク片をその位置に固定表示

3. 保持時間を過ぎた
   → ピーク片が徐々に下降

4. バーが再上昇してピークに追いついた
   → 現在値に吸収され、新しいピークとして扱う
```

ピークホールドは、通常バー、デジタルバー、片側バー、両側バー、スクロール、グラデーション、マット出力で共通して使えます。

### ピークホールド

ON / OFF を切り替えます。

基本プリセットではOFF、派手系・デジタル系プリセットではONになっています。

### 保持ms

ピーク片を固定表示する時間です。

標準は `100ms` です。短くすると軽快になり、長くするとピーク位置が強調されます。

### 下降ms

保持時間後、ピーク片が上端から下端まで落ちる目安時間です。

標準は `300ms` です。短くすると素早く落ち、長くするとゆっくり残ります。

### ピーク長%

通常バーでのピーク片の長さです。最大バー長に対する割合で指定します。

標準は `4%` です。実際の描画では、見えやすさのため 2〜10px 程度に収まるよう調整されます。

### デジタルピーク区画

デジタル表示時のピーク片の区画数です。

通常は `1` で十分です。太くしたい場合だけ `2` 以上にします。

### マット出力との関係

メイン動画にピーク片が出る場合、マット動画にも同じピーク片の形状が黒で出ます。

これにより、比較合成時にピーク片だけが背景に負けることを避けられます。

## Post Transform layer

Post Transform layer は、バーやピーク片をすべて描画したあとの1フレーム画像全体に対して、後段で座標変換をかける層です。

従来の出力は、Post Transform が `none` の状態、つまり恒等写像として扱われます。

```text
従来:
(x, y) -> (x, y)

Post Transform:
(x, y) -> (x', y')
```

この層により、通常バー、デジタルバー、ピークホールド、グラデーション、スクロールなどを描き終えたあとで、完成画像全体を変形できます。

### 重要な考え方

- 変形は完成した1フレーム画像に対して行います。
- バー描画ロジックの内部には、回転や台形変換の分岐を入れていません。
- メイン動画とマット動画には同じ Post Transform を適用します。
- キャンバスサイズは維持します。
- はみ出した部分はクリップします。
- 空いた領域は、メイン動画では黒、マット動画では白で埋めます。

### 回転

「回転角度°」で、描画済みフレーム全体を中心まわりに回転します。

`0` なら従来どおりです。右側の「リセット」ボタンで `0` に戻せます。

### 縦台形

「縦台形」は、上辺または下辺を狭める変形です。

- マイナス: 上部が小さくなります。
- `0`: 変形なしです。
- プラス: 下部が小さくなります。

### 横台形

「横台形」は、左辺または右辺を狭める変形です。

- マイナス: 左側が小さくなります。
- `0`: 変形なしです。
- プラス: 右側が小さくなります。

### 台形 + 回転

回転と台形変換は排他ではありません。同時に使えます。

内部的には、長方形の完成フレームを1枚の平面として台形に写像し、その形を保ったまま回転するように扱います。そのため、「台形変換されたスペアナ素材が、そのまま斜めに回転している」見た目になります。

### 音量連動の拡大

音量連動の拡大も Post Transform の一部です。

ONにすると、フレームごとの音量から拡大率を求め、完成フレーム全体に拡大または縮小を適用します。

`音量拡大率%` は `80～150%` です。100%より大きい値では音が大きいと拡大し、100%より小さい値では音が大きいと縮小します。

拡大率は、次の流れで決まります。

```text
表示バー値
-> 必要なら低域側だけを抽出
-> 平均音量を計算
-> 拡大開始閾値 / 最大拡大天井で正規化
-> 音量保持ms / 音量減衰msでピークホールド風に整形
-> 100% と音量拡大率% の間で補間
```

「低域のみ参照」がONの場合、低域側のバーだけを使って音量を判定します。バスドラムやベースに合わせて拍動させたい場合に向きます。

### メイン動画・マット動画との関係

Post Transform は、メイン動画とマット動画の両方に同じ設定で適用されます。

空いた領域は、メイン動画では黒、マット動画では白で埋められます。

## 生成・動きタブ

生成・動きタブでは、数値ではなく定性的な選択肢でスペアナの動きを調整します。

### 反応の速さ

バーが音の変化にどれくらい素早く反応するかを指定します。

- ゆったり
- 標準
- 速い
- カスタム

内部的には、FFTサイズ、Attack、Release などに影響します。

### 上下動の強さ

ピーク後にどの程度下まで落ちるか、上下動をどの程度大きく見せるかを指定します。

- 控えめ
- 標準
- 大きい
- カスタム

### 感度

音源に対する全体的な反応感度です。

- 低め
- 標準
- 高め
- カスタム

バーが低すぎる場合は高めに、常に高すぎる場合は低めにします。

### 動きの細かさ

バーの動きの細かさです。

- なめらか
- 標準
- 細かい
- カスタム

### 高低差の出方

バー同士の高さの差をどの程度強く出すかを指定します。

- フラット
- 標準
- メリハリ
- カスタム

`メリハリ` にすると帯域ごとの差が強く見えます。`フラット` にすると全体がなだらかに見えます。

### 動きプレビューに適した区間を自動検出

ONにすると、音源の先頭付近をスキャンし、動きプレビューに適した区間を自動的に選びます。

これは GUI 内の動きプレビュー用です。動画プレビュー MP4 は、動画編集ソフトのタイムライン先頭で試すため、常に音源の先頭から生成されます。

### 手動開始秒

自動検出をOFFにした場合、動きプレビューを何秒地点から始めるかを指定します。

### 動画プレビュー秒数

「30秒プレビュー動画生成」で作るプレビューMP4の長さです。

### ウォームアップ秒

動きプレビュー開始位置より少し前から解析する秒数です。

曲中からプレビューする場合、直前の音の状態を少し考慮することで、全体生成時に近い動きになります。

## 詳細設定タブ

詳細設定タブでは、内部パラメータを直接変更できます。通常は触らなくても構いません。

詳細設定を変更すると、生成・動きタブの定性的設定は `カスタム` 扱いになります。

### 背景色

通常の素材出力で使う背景色です。

現在の標準ワークフローでは、通常は黒のまま使います。ペア出力時のマット動画では、自動的に白背景になります。

### 低域参照範囲%

「音量連動の拡大」で「低域のみ参照」がONのとき、低域側から何%のバーを音量判定に使うかを指定します。

標準は `25%` です。自動周波数範囲がたとえば 60～5000Hz 付近の場合、低域側25%はバスドラムやベース寄りの成分を拾いやすい目安になります。

### 周波数設定

- 自動
- 手動

自動では、音源の先頭部分を解析し、実用的な周波数範囲を推定します。

手動では、最低周波数Hzと最高周波数Hzを直接指定します。

周波数設定が「自動」の場合、解析が終わるとコンボボックスの横に `自動設定音域: xx～xxHz` が表示されます。音源ロード直後ではなく、プレビューや生成で解析が走ったあとに表示されます。

### sample rate

内部解析用のサンプルレートです。通常は `24000` のままで十分です。

### FFT

周波数解析の窓サイズです。

小さい値は反応が速くなりやすく、大きい値は滑らかになりやすいです。

### 内部解析バー数

内部で解析する帯域数です。

表示バー数とは別です。内部解析バー数を高めにしておくことで、表示バー数を変えても動きの印象を安定させやすくなります。

### min dB / max dB

絶対音量ゲートの下限・上限です。

無音やほぼ無音でバーが残る場合は、`min dB` や `Base cut` を調整すると改善することがあります。

### Attack / Release

バーが上がるとき、下がるときの追従係数です。

### Relative range dB

各フレームのピークから、どの範囲をバー高さに変換するかを指定します。

### Peak %

相対正規化でピーク扱いするパーセンタイルです。

### Base cut

相対正規化後の低い値を切る割合です。

値を上げると、低いバーがより下まで戻りやすくなります。

### Pulse amount / Pulse speed

持続音をそのまま張り付かせず、新しいピークを強調するための設定です。

### Gamma

バー高さの見た目補正です。

1より小さいと、低めの値でもバーが高く見えます。1より大きいと、低めの値が抑えられます。

### CRF

MP4の画質設定です。

小さいほど高画質・大容量です。標準は18です。

### Encoder

ffmpeg の映像エンコーダーです。

標準は `libx264` です。環境によっては `h264_nvenc` なども使えますが、対応していない環境では失敗します。

### x264 preset

`libx264` の速度・圧縮設定です。標準は `veryfast` です。

## 30秒プレビュー動画生成

「30秒プレビュー動画生成」は、短いMP4ファイルを出力します。

用途:

- 動画編集ソフトに読み込めるか確認する。
- 比較合成用ペア出力の重なり方を確認する。
- バーの高さ・幅・位置を確認する。
- 色やピークホールドの見え方を確認する。
- エンコード設定が動作するか確認する。

このプレビューMP4は、常に音源の先頭から生成されます。

GUI内の動きプレビューとは目的が違います。GUI内の動きプレビューは、自動検出された曲中の盛り上がり部分を使うことがあります。

---

## 全体生成

「全体生成」は、音源全体に対応するスペアナMP4を生成します。

長尺音源にも対応します。実際の処理時間は、PC性能、音源の長さ、fps、解像度、エンコード設定によって変わります。

ペア出力ONの場合、v1.3.0ではメイン動画とマット動画を並列に生成します。メインとマットは同じ解析結果、同じピークホールド、同じ Post Transform を使うため、フレーム形状は一致します。

回転、台形変換、音量連動拡大、デジタル表示、スクロール、ピークホールドを同時に使う設定は重くなります。これはPost Transformの柔軟性と引き換えの処理コストです。

60分、30fps の動画では 108,000 フレームになります。まず30秒プレビューで見た目と合成を確認してから全体生成することをおすすめします。

---

## 出力ファイル名と上書き防止

既存ファイルは上書きしません。

通常出力では、必要に応じて ` (1)`, ` (2)` のように番号を付けます。

ペア出力では、メイン動画とマット動画を1組として扱います。どちらか一方でも既存ファイルと重複する場合、両方に同じ番号を付けます。

例:

```text
song_spectrum.mp4
song_spectrum_matte_dark.mp4
```

どちらかが既に存在する場合:

```text
song_spectrum (1).mp4
song_spectrum (1)_matte_dark.mp4
```

これにより、メインだけ元名、マットだけ連番、という分かりにくい状態を避けます。

## 動画編集ソフトでの使い方

### ペア出力を使う場合

推奨する重ね方です。

```text
1. 背景動画または背景画像
2. マット動画: 比較（暗） / Darken
3. メイン動画: 比較（明） / Lighten
```

この順番で重ねると、明るい背景でもスペアナがきれいに乗りやすくなります。

### 通常のメイン動画だけを使う場合

白バー、黒背景のシンプルな素材なら、メイン動画だけでも使えます。

```text
1. 背景動画または背景画像
2. メイン動画: 比較（明） / Lighten / Screen
```

背景が暗めの場合はこれでも十分です。背景が明るい場合や、色付きバー、ピークホールドを使う場合は、ペア出力を推奨します。

## 推奨ワークフロー

### まず安全に作る

1. `01 Basic White` を選ぶ。
2. ペア出力ONのまま30秒プレビューを作る。
3. 動画編集ソフトで、マットを比較（暗）、メインを比較（明）で重ねる。
4. 問題なければ全体生成する。

### 派手な表示にしたい

1. `04 Neon Dual` または `09 Digital Neon Scroll` を選ぶ。
2. 動きプレビューでピークホールドの残り方を確認する。
3. 必要に応じて「保持ms」「下降ms」を調整する。
4. 30秒プレビューで合成確認する。

### LEDメーター風にしたい

1. `07 Digital LED` または `08 Digital Dual` を選ぶ。
2. 分割数、ギャップpx、デジタルピーク区画を調整する。
3. デジタルピーク区画はまず1のまま使う。
4. 太く見せたい場合だけ2にする。

### 長尺BGMに使う

1. `06 Smooth Ambient` を選ぶ。縦型や投影風の見た目にしたい場合は `11 Vertical Scroll` や `10 Trapezoid Neon` も候補になります。
2. 最大高さを少し下げる。
3. ピークホールドは必要な場合だけONにする。
4. 30秒プレビューで邪魔にならないか確認する。
5. 全体生成する。

### SRT Spectrum Video Composer で最終動画化する

1. Audio Spectrum Overlay Maker でメイン動画とマット動画を生成する。
2. 「SRT Spectrum Video Composer を開く」を押す。
3. Composer 側で背景画像、SRT字幕、出力MP4を指定する。
4. Composer 側で最終動画を生成する。

## トラブルシューティング

### アプリが起動しない

Python 3.10 以上が `py -3` で起動できるか確認してください。

```bat
py -3 --version
```

依存パッケージが入っていない場合は、`setup.bat` を実行してください。

### ffmpeg が見つからない

`ffmpeg.exe` と `ffprobe.exe` をアプリ所在地の `bin` フォルダに置くか、PATH に入れてください。

確認:

```bat
ffmpeg -version
ffprobe -version
```

### 音源ファイルが読み込めない

ffmpeg がその形式を読めるか確認してください。特殊なコーデックや壊れたファイルでは失敗することがあります。

MP3 や WAV に変換してから試すと安定する場合があります。

### 30秒プレビューが静かなイントロだけになる

仕様です。

30秒プレビューMP4は、動画編集ソフトのタイムライン先頭で試すため、常に音源の先頭から生成します。

曲中の盛り上がりで動きを確認したい場合は、GUI内の「動きプレビュー」を使ってください。

### バーが低すぎる

「感度」を「高め」にしてください。

それでも低い場合は、詳細設定の `gain_db` を上げます。

### バーが高すぎる

「感度」を「低め」にしてください。

それでも高い場合は、詳細設定の `gain_db` を下げます。

### 動きが鈍い

「反応の速さ」を「速い」にするか、「動きの細かさ」を「細かい」にしてください。

### 動きが暴れすぎる

「反応の速さ」を「ゆったり」にするか、`06 Smooth Ambient` を使ってください。

### ピークホールドが目立ちすぎる

次を試してください。

- ピークホールドをOFFにする。
- 保持msを短くする。
- 下降msを短くする。
- ピーク長%を下げる。
- デジタルピーク区画を1に戻す。

### ピークホールドが見えにくい

次を試してください。

- 保持msを長くする。
- 下降msを長くする。
- ピーク長%を上げる。
- デジタルピーク区画を2にする。
- バー色1/2を明るくする。

### 動画編集ソフトでうまく重ならない

まずはペア出力を使ってください。

```text
マット動画: 比較（暗）
メイン動画: 比較（明）
```

メイン動画だけを比較（明）で重ねるより、明るい背景で安定しやすくなります。

### 色付きバーが思った通りに見えない

動画編集ソフトのブレンドモードによって見え方が変わります。

まず白→白の単色で合成確認し、その後に色を付けるのが安全です。

### SRT Spectrum Video Composer が開かない

`final_composer.py` が `app.py` または EXE と同じフォルダにあるか確認してください。

同梱版 ZIP では、通常は最初から同じ階層に入っています。

PyInstaller で EXE 化した場合は、`final_composer.py` を `--add-data` で同梱するか、EXE と同じフォルダに置いてください。アプリは PyInstaller の展開先、EXE横、`app.py` 横の順で探します。

### SRT Spectrum Video Composer にファイルが渡らない

Audio Spectrum Overlay Maker でまだ音源を選んでいない、またはスペアナ動画を生成していない場合、handoff JSON の該当項目は空欄で渡されます。

Composer 側で手動選択してください。

### h264_nvenc で失敗する

GPUやffmpegビルドが NVENC に対応していない可能性があります。

標準の `libx264` に戻してください。

### 生成ファイルが大きい

CRFを上げるとファイルサイズが小さくなります。

標準は18です。20〜23程度でも、この用途では問題ない場合があります。

### 音声とスペアナがシンクロしない

次を確認してください。

- 元音源とスペアナMP4を同じ開始位置に置いているか。
- スペアナMP4を動画編集ソフト側で伸縮していないか。
- 音源を別途編集して長さが変わっていないか。
- タイムラインのfps変換でズレていないか。

Audio Spectrum Overlay Maker の全体生成は、音源長に対応するフレームを生成します。動画編集ソフト側で時間伸縮しなければ、基本的には最後までシンクロします。

## 内部処理の考え方

内部的には、次の流れで動画を作ります。

```text
音源ファイル
↓
ffmpeg でモノラル音声にデコード
↓
FFT による周波数解析
↓
Dynamic 処理
↓
内部解析値 SpectrumData を作成
↓
表示バー数に変換
↓
ピークホールド値を生成
↓
フレームごとにバー本体とピーク片を描画
↓
Post Transform
↓
ffmpeg にフレームを渡して MP4 エンコード
```

### 解析層

音源から内部解析値 `SpectrumData` を作ります。

`SpectrumData` には、Dynamic処理後の値、処理前のdB値、周波数境界、fps、sample rate、内部解析バー数、周波数範囲などが含まれます。

### 変形層

内部解析値を、実際に表示するバー本数へ変換します。

表示バー数と内部解析バー数を分けることで、バー本数を変えても動きの印象が大きく変わりにくくなっています。

### ピークホールド層

表示用 `bar_values` から、同じ形の `peak_values` を作ります。

この `peak_values` はメイン動画とマット動画で共有されます。そのため、ピーク片の位置と形状がフレーム単位で一致します。

### 描画層

変形済みの値をフレーム画像に描画します。

現在の描画は、通常バー、デジタルバー、片側バー、両側バー、ピークホールドに対応しています。

### Post Transform層

描画済みのRGBフレーム全体に対して、後段で座標写像を適用します。

この層は、バー本体やピーク片の描画ロジックとは分離されています。`none` の場合は恒等写像として扱われ、従来出力と同じ見た目になります。

メイン動画とマット動画には同じPost Transformが適用されます。

### エンコード層

描画済みフレームを ffmpeg の rawvideo pipe に渡し、MP4 としてエンコードします。

出力MP4には音声を含めません。

## 制限事項

現在の主な制限は次の通りです。

- 真の透過MP4は出力しません。
- 音声入りMP4は出力しません。
- 透過PNG連番は出力しません。
- 円形スペアナは未対応です。
- 複数音源の一括処理は未対応です。
- 数時間級の音源では、メモリ使用量と処理時間が大きくなる可能性があります。

真の透過MP4やPNG連番ではなく、メイン動画 + マット動画の比較合成方式を採用しているのは、一般的な動画編集ソフトで扱いやすく、長尺動画でも現実的に管理しやすいためです。
