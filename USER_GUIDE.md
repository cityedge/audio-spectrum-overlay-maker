# Audio Spectrum Overlay Maker v1.0.0 User Guide

Audio Spectrum Overlay Maker は、音源ファイルを解析し、音に連動して上下するスペクトラムアナライザー風の MP4 オーバーレイ動画を生成するローカルアプリです。

主な用途は、AI生成曲・オリジナル曲・アルバム動画・歌詞動画・配信用動画に、黒背景＋白いスペアナバーの素材を重ねることです。動画編集ソフト側で「比較（明）」「Lighten」「Screen」などの合成モードを使うと、黒背景を実質的に消し、白いバーだけを画像や動画の上に重ねられます。

This app is a local Python/Tkinter tool for generating audio-reactive spectrum-analyzer overlay MP4 videos. It analyzes an audio file locally and writes a silent MP4 overlay video.

## このアプリでできること

Audio Spectrum Overlay Maker では、次のことができます。

- 音源ファイルを読み込み、音量・周波数成分に反応するスペアナ動画を生成する。
- 黒背景・白バーの横型スペアナ素材を生成する。
- 出力動画の横幅・高さ・fps・バー本数・バー幅・角丸・余白などを調整する。
- 実際の音源を使って、GUI内で短い「動きプレビュー」を再生する。
- 動画編集ソフトに読み込むための30秒プレビューMP4を生成する。
- 55分程度の長尺音源でも、音源と最後までシンクロするMP4を生成する。
- システムプリセットやユーザープリセットを使って、設定を保存・再利用する。
- 通常ユーザー向けには定性的な「生成・動き」設定を使い、上級者は「詳細設定」で内部パラメータを直接調整する。
- 日本語 / English のインターフェイスを切り替える。

v1.0.0 は、公式版としてまず「平面・横型・通常バー」のスペアナに絞っています。円形スペアナ、面グラフ、点表示、スクロール、グラデーションなどは、今後の拡張対象です。

## このアプリが出力するもの

出力されるのは、**音声なしの MP4 動画**です。

音源ファイルは、あくまでスペアナの動きを作るための解析用です。生成されたMP4には音声は入りません。したがって、動画編集ソフトでは、次のように使います。

1. 元の音源を音声トラックとして配置する。
2. アルバムアートや背景画像を映像トラックとして配置する。
3. Audio Spectrum Overlay Maker で生成したスペアナMP4を別の映像トラックに配置する。
4. スペアナMP4に「比較（明）」「Lighten」「Screen」などを適用する。
5. 白いスペアナバーだけが背景の上に重なる。

標準設定では、黒背景に白いバーが描かれます。黒背景は動画編集ソフト側の合成で消しやすく、白いバーは視認性が高いため、最も安定した基本形です。

## 処理の流れ

内部的には、次の流れで動画を作ります。

```text
音源ファイル
↓
ffmpeg でモノラル音声にデコード
↓
FFT による周波数解析
↓
Dynamic 処理
↓
内部解析値 SpectrumData を作成
↓
表示バー数に変換
↓
フレームごとにバーを描画
↓
ffmpeg にフレームを渡して MP4 エンコード
```

v1.0.0 では、解析層と変形層を分けています。これにより、見た目のバー数を18/24/32/48/64に変えても、動きの心理的な印象が大きく変わりにくくなっています。

## 動作環境

推奨環境は次の通りです。

- Python 3.10 以降
- `ffmpeg` と `ffprobe` が PATH に入っていること
- `numpy`
- `Pillow`
- Tkinter が使える Python 環境

Windows では、Python 公式版に Tkinter が含まれていることが一般的です。

ffmpeg は、音声の読み込みと MP4 のエンコードに使います。`ffmpeg` だけでなく `ffprobe` も PATH 上に必要です。

## インストールと起動


### Windows用バッチファイルで起動する場合

Windowsでは、同梱の `setup.bat` と `start.bat` を使えます。

#### setup.bat

初回セットアップ用です。

実行内容：

- Python launcher `py` が存在するか確認する。
- `ffmpeg` が PATH 上にあるか確認する。
- `ffprobe` が PATH 上にあるか確認する。
- `.venv` がなければ作成する。
- `pip` を更新する。
- `requirements.txt` の依存パッケージをインストールする。

初回のみ、`setup.bat` をダブルクリックしてください。

#### start.bat

アプリ起動用です。

実行内容：

- `.venv\Scripts\python.exe` があるか確認する。
- `.venv` がなければ `setup.bat` を自動実行する。
- `.venv` 内の Python で `app.py` を起動する。

通常は、2回目以降は `start.bat` をダブルクリックするだけで起動できます。


### 1. ファイルを展開する

GitHub からダウンロードしたZIPを展開します。任意のフォルダで構いません。

### 2. Python依存パッケージを入れる

コマンドプロンプトまたはターミナルで、展開したフォルダに移動し、次を実行します。

```bash
pip install -r requirements.txt
```

Windows で `pip` が見つからない場合は、次を試してください。

```bat
py -m pip install -r requirements.txt
```

### 3. ffmpeg を準備する

`ffmpeg` と `ffprobe` が PATH 上にある必要があります。

確認例：

```bash
ffmpeg -version
ffprobe -version
```

どちらもバージョン情報が表示されれば、基本的には利用できます。

### 4. アプリを起動する

```bash
python app.py
```

Windows の場合：

```bat
py app.py
```

## 音源と出力先

### 音源

`音源` フォームには、解析する音源ファイルを指定します。

対応形式は、基本的にローカルの ffmpeg が読み込める音声形式です。例：

- WAV
- MP3
- M4A
- AAC
- FLAC
- MP4 内の音声

ただし、実運用では WAV / MP3 / M4A が扱いやすいです。

### 出力先

`出力先` フォームには、生成した MP4 を保存するフォルダを指定します。

未指定の場合は、音源ファイルと同じフォルダに保存されます。

## プリセット

プリセットは、右ペインの「見た目」「生成・動き」「詳細設定」に関係する設定を保存・呼び出しする機能です。

### システムプリセット

v1.0.0 には、次のシステムプリセットがあります。

- `01 Dynamic Standard`
- `02 Dynamic Active`
- `03 Dynamic Smooth`

システムプリセットは削除できません。  
また、システムプリセットと同じ名前でユーザープリセットを保存することもできません。

### 01 Dynamic Standard

標準設定です。まずはこれを基準に使うことを推奨します。

特徴：

- 24本バー
- 黒背景
- 白バー
- 自動周波数設定
- 標準的な反応速度
- 標準的な上下動
- 標準的な感度

### 02 Dynamic Active

より動きを強めた設定です。ピーク後に下まで戻りやすく、上下動が大きく見えます。

向いている用途：

- EDM
- アップテンポなJ-pop
- 明るめの楽曲
- 動きの目立つ動画素材

### 03 Dynamic Smooth

動きを少し落ち着かせた設定です。長尺動画や、背景の邪魔をしたくない用途に向きます。

向いている用途：

- Lo-fi
- バラード
- 長尺アルバム動画
- 作業用BGM的な動画

### プリセット保存

現在の設定を保存するには、`プリセット保存` を押します。  
名前を入力すると、ユーザープリセットとして保存されます。

すでに存在するユーザープリセット名を指定した場合は、上書き確認が出ます。

### プリセット削除

ユーザープリセットを選んだ状態で `プリセット削除` を押すと、そのプリセットを削除できます。

システムプリセットは削除できません。

### カスタム状態

プリセットを選んだあと、設定を変更すると、プリセット欄は `カスタム` になります。

これは、現在の設定が選択中のプリセットと一致しなくなったことを示します。

### 保存されるもの

プリセットには、出力されるスペアナ動画に影響する設定が保存されます。

保存対象の例：

- 横幅
- 高さ
- fps
- バー本数
- 背景色
- バー色
- バー幅
- 角丸
- 余白
- 生成・動きの定性的設定
- 詳細設定の値
- エンコード設定

### 保存されないもの

作業中だけ使う値は保存されません。

保存されないものの例：

- 音源ファイル
- 出力先フォルダ
- 言語設定
- 動きプレビューの開始秒
- 動きプレビューの秒数
- 30秒プレビュー動画の秒数
- 自動検出のスキャン秒数
- ログ内容

これにより、プリセットは「動画の見た目・動き」を保存するためのものとして扱われます。

## プレビュー

### 見た目プレビュー

左上のプレビューです。  
現在の見た目設定を使い、静止状態のスペアナ素材を表示します。

確認できるもの：

- 画面サイズ
- バー本数
- バー幅
- バー色
- 背景色
- 角丸
- 左右余白
- 下余白
- 最大高さ

設定を変更すると、自動的に更新されます。  
従来あった「見た目更新」ボタンは不要になったため、v1.0.0では削除しています。

数値を手入力で編集している途中に、欄が一時的に空になったり、不正な色コードになったりしても、見た目プレビューにエラーメッセージは表示しません。入力途中の一時的な無効値は無視します。

### 動きプレビュー

左下のプレビューです。  
実際の音源を短く解析し、スペアナの動きをGUI内で再生します。

`動きプレビュー` ボタンを押すと、音源の一部を解析して一度だけ再生します。

`停止` ボタンを押すと、再生中の動きプレビューを停止します。

動きプレビューは確認用です。  
本番のMP4ファイルは、下部の `30秒プレビュー動画生成` または `全体生成` で作成します。

## 見た目タブ

見た目タブでは、生成されるスペアナ素材の外観を調整します。

### バー本数

表示するバーの本数です。

選択肢：

- 18
- 24
- 32
- 48
- 64

少ない本数ほどシンプルで太く見えます。  
多い本数ほど細かく、密度の高いスペアナになります。

v1.0.0では、表示バー数と内部解析バー数を分離しているため、バー本数を変えても、動きの速度感が大きく変わりにくくなっています。

### 背景色

素材動画の背景色です。標準は黒です。

標準用途では、黒背景のまま使うことを推奨します。  
動画編集ソフトで比較（明）/ Lighten / Screen 系の合成を使うと、黒背景を実質的に消せます。

### バー色

バーの色です。標準は白です。

白バーは最も合成しやすく、視認性も高いです。

### バー幅%

各バーの幅です。

値を下げると、バーが細くなり、バー同士の間隔が広がります。  
値を上げると、バーが太くなり、密度が高く見えます。

バーの中心位置は変えずに、太さだけを変えます。

### 角丸半径px

バーの角丸半径です。

値を大きくすると、バーがカプセル状に近づきます。  
v1.0.0では、小音量時にバーがフル幅の四角い小片として出ないよう、角丸バーでは中心から小さなレモン/ピル形状として出現し、高さが増えるにつれて通常幅へ広がる描画にしています。

### 最大高さ%

バーが使用する最大高さです。

値を大きくすると、ピーク時のバーが高くなります。  
値を小さくすると、バー全体が低めに抑えられます。

### 左右余白%

スペアナ全体の左右余白です。

値を上げると、左右の余白が広くなり、スペアナ全体が中央に寄ります。

### 下余白%

バーの基準線から下端までの余白です。

値を上げると、バー全体が少し上に移動します。

### 横幅

生成するスペアナ素材の横幅です。

### 高さ

生成するスペアナ素材の高さです。

### fps

生成する動画のフレームレートです。

選択肢：

- 24
- 30
- 60

標準は30fpsです。通常は30fpsで十分です。

## 生成・動きタブ

生成・動きタブでは、数値ではなく、定性的な選択肢でスペアナの動きを調整します。

### 反応の速さ

バーが音の変化にどれくらい素早く反応するかを指定します。

日本語：

- ゆったり
- 標準
- 速い
- カスタム

English：

- Slow
- Standard
- Fast
- Custom

内部的には、FFTサイズ、Attack、Release などに影響します。

### 上下動の強さ

ピーク後にどの程度下まで落ちるか、上下動をどの程度大きく見せるかを指定します。

日本語：

- 控えめ
- 標準
- 大きい
- カスタム

English：

- Subtle
- Standard
- Large
- Custom

内部的には、Relative range、Base cut、Pulse amount、Pulse speed などに影響します。

### 感度

音源に対する全体的な反応感度です。

日本語：

- 低め
- 標準
- 高め
- カスタム

English：

- Low
- Standard
- High
- Custom

バーが低すぎる場合は高めに、常に高すぎる場合は低めにします。

### 動きの細かさ

バーの動きの細かさです。

日本語：

- なめらか
- 標準
- 細かい
- カスタム

English：

- Smooth
- Standard
- Detailed
- Custom

内部解析バー数やFFT設定に影響します。

### 高低差の出方

バー同士の高さの差をどの程度強く出すかを指定します。

日本語：

- フラット
- 標準
- メリハリ
- カスタム

English：

- Flat
- Standard
- High Contrast
- Custom

`メリハリ` にすると、帯域ごとの差が強く見えます。  
`フラット` にすると、全体がなだらかに見えます。

### 詳細カスタム状態

詳細設定タブで内部パラメータを直接変更すると、生成・動きタブの定性的コンボボックスは `カスタム` 扱いになり、ロックされます。

これは、詳細設定の数値が優先されている状態で、一部の定性的項目だけを変更して不整合が起こるのを防ぐためです。

定性的設定に戻すには、詳細設定タブの `定性的設定から再計算` を押してください。

### 動きプレビューに適した区間を自動検出

オンにすると、音源の先頭付近をスキャンし、動きプレビューに適した区間を自動的に選びます。

静かなイントロが長い曲でも、動きのある場所を使って確認しやすくなります。

この機能はGUI内の動きプレビュー用です。  
本番の30秒プレビューMP4は、動画編集ソフトのタイムライン先頭で試すため、常に音源の先頭から生成されます。

### 手動開始秒

`動きプレビューに適した区間を自動検出` がオフのとき、動きプレビューを何秒地点から始めるかを指定します。

### 動きプレビュー秒数

GUI内で再生する動きプレビューの長さです。

### 動画プレビュー秒数

`30秒プレビュー動画生成` で作るプレビューMP4の長さです。  
標準は30秒です。

### ウォームアップ秒

動きプレビュー開始位置より少し前から内部解析する秒数です。

曲中からプレビューする場合、直前の音の状態を少し考慮することで、全体生成時に近い動きになります。

## 詳細設定タブ

詳細設定タブでは、内部パラメータを直接変更できます。通常は触らなくても構いません。

詳細設定を1つでも変更すると、`詳細カスタム` 状態になります。この状態では、生成・動きタブの定性的設定はロックされ、詳細設定の値が優先されます。

### 周波数設定

選択肢：

- 自動 / Auto
- 手動 / Manual

`自動` では、音源の先頭部分を解析し、低域・高域の周波数範囲を推定します。

`手動` では、最低周波数Hz、最高周波数Hzを直接指定します。

### 最低周波数Hz

手動設定時の最低周波数です。

低くすると、低音側の成分を含みやすくなります。  
ただし、低すぎると不要な低域ノイズを拾うことがあります。

### 最高周波数Hz

手動設定時の最高周波数です。

高くすると、ハイハットやシンバルなど高域成分まで含みやすくなります。  
ただし、高すぎると細かいノイズ成分も反応しやすくなります。

### 定性的設定から再計算

詳細カスタム状態を解除し、生成・動きタブの定性的設定から詳細パラメータを再計算します。

### 詳細カスタム

詳細設定が優先されていることを示すチェックです。

### sample rate

内部解析用のサンプルレートです。

通常は `24000` のままで十分です。  
高くすると高域解析の上限が上がりますが、処理負荷も上がります。

### FFT

周波数解析の窓サイズです。

小さい値は反応が速くなりやすく、大きい値は滑らかになりやすいです。

選択肢：

- 512
- 1024
- 2048
- 4096

### 内部解析バー数

内部で解析する帯域数です。

表示バー数とは別です。  
内部解析バー数を高めにしておくことで、表示バー数を変えても動きの印象を安定させやすくなります。

選択肢：

- 48
- 64
- 96
- 128

### min dB

絶対音量ゲートの下限です。

v1.0.0では、無音・ほぼ無音の区間でバーが残らないよう、min dB 以下ではゲートが完全に0になります。

### max dB

絶対音量ゲートの上限です。

### Attack

バーが上がるときの追従係数です。

1.0に近いほど素早く上がります。

### Release

バーが下がるときの追従係数です。

1.0に近いほど素早く下がります。

### Relative range dB

各フレームのピークから、どの範囲をバー高さに変換するかを指定します。

小さいほど、ピーク付近が強く出やすくなります。  
大きいほど、広い範囲がなだらかに表示されます。

### Peak %

相対正規化でピーク扱いするパーセンタイルです。

通常は90前後で十分です。

### Base cut

相対正規化後の低い値を切る割合です。

値を上げると、低いバーがより下まで戻りやすくなります。

### Pulse amount

持続音をいったん落とし、新しいピークを強調する量です。

Dynamicらしい上下動の中心的な設定です。

### Pulse speed

持続音を基準値として吸収する速さです。

値を上げると、ピーク後の落ち込みが強くなります。

### Gamma

バー高さの見た目補正です。

1より小さいと、低めの値でもバーが高く見えます。  
1より大きいと、低めの値がより抑えられます。

### CRF

MP4の画質設定です。

小さいほど高画質・大容量です。  
標準は18です。

### Encoder

ffmpeg の映像エンコーダーです。

標準は `libx264` です。  
環境によっては `h264_nvenc` なども使えますが、対応していない環境では失敗します。

### x264 preset

`libx264` の速度・圧縮設定です。

標準は `veryfast` です。  
この用途では、`veryfast` でも十分な品質が得られます。

## 30秒プレビュー動画生成

`30秒プレビュー動画生成` は、短いMP4ファイルを出力します。

用途：

- 動画編集ソフトに読み込めるか確認する
- 合成モードを確認する
- バーの高さ・幅・位置を確認する
- ファイルサイズ感を確認する
- エンコード設定が動作するか確認する

このプレビューMP4は、常に音源の先頭から生成されます。  
これは、動画編集ソフトのタイムライン先頭に置いて確認しやすくするためです。

GUI内の動きプレビューとは別です。GUI内の動きプレビューは、自動検出された曲中の盛り上がり部分を使うことがあります。

---

## 全体生成

`全体生成` は、音源全体に対応するスペアナMP4を生成します。

長尺音源にも対応します。  
実際の処理速度はPC性能、音源の長さ、fps、解像度、エンコード設定によって変わります。

例として、55分程度の音源を64本バーで生成しても、ffmpegとPC環境が適切であれば実用的な時間で生成できます。

---

## 出力ファイル名と上書き防止

出力ファイルがすでに存在する場合、上書きはしません。

代わりに、次のように連番を付けて保存します。

```text
song_spectrum_720x280_64bars_w62.mp4
song_spectrum_720x280_64bars_w62 (1).mp4
song_spectrum_720x280_64bars_w62 (2).mp4
```

これにより、試行錯誤中に以前の出力を誤って消すことを防ぎます。

## 動画編集ソフトでの使い方

標準的な使い方は次の通りです。

1. 動画編集ソフトに背景画像またはアルバムアートを配置する。
2. 同じタイムラインに元の音源を配置する。
3. Audio Spectrum Overlay Maker で生成したMP4を、上位の映像トラックに配置する。
4. スペアナMP4の合成モードを、比較（明）/ Lighten / Screen 系にする。
5. 黒背景が消え、白いスペアナバーだけが見える状態にする。
6. 位置やサイズを調整する。
7. 必要なら不透明度を下げる。

PowerDirectorでは、「比較（明）」系の合成が有効です。  
他の編集ソフトでも、同等の Lighten / Screen / Add 系の合成が使える場合があります。

## 推奨ワークフロー

### 基本ワークフロー

1. 音源を選択する。
2. 出力先を選択する。
3. `01 Dynamic Standard` を選ぶ。
4. 見た目タブでバー本数、バー幅、高さ、余白を調整する。
5. `動きプレビュー` を押して動きを確認する。
6. `30秒プレビュー動画生成` を押す。
7. 動画編集ソフトに読み込んで合成を確認する。
8. 問題なければ `全体生成` を押す。

### 長尺アルバム動画のワークフロー

1. アルバム全体の音源を用意する。
2. まず30秒プレビューで合成確認を行う。
3. 必要に応じてバー本数を48または64にする。
4. 動画編集ソフトで位置・サイズを決める。
5. 全体生成する。
6. 生成されたスペアナMP4を、アルバム音源と同じ開始位置に配置する。
7. 最後までシンクロしているか確認する。

### 落ち着いた表示にしたい場合

- `03 Dynamic Smooth` を使う。
- バー本数を24または32にする。
- 最大高さを下げる。
- バー幅を少し下げる。
- 不透明度を動画編集ソフト側で下げる。

### 動きを目立たせたい場合

- `02 Dynamic Active` を使う。
- バー本数を48または64にする。
- 最大高さを上げる。
- 高低差の出方を「メリハリ」にする。
- 上下動の強さを「大きい」にする。

## トラブルシューティング

### アプリが起動しない

Pythonが正しくインストールされているか確認してください。

```bash
python --version
```

Windowsの場合：

```bat
py --version
```

依存パッケージを入れていない場合は、次を実行してください。

```bash
pip install -r requirements.txt
```

### ffmpeg が見つからない

`ffmpeg` と `ffprobe` が PATH に入っている必要があります。

確認：

```bash
ffmpeg -version
ffprobe -version
```

どちらかが見つからない場合、音源の読み込みやMP4の生成ができません。

### 音源ファイルが読み込めない

ffmpegがその形式を読めるか確認してください。  
特殊なコーデックや壊れたファイルでは失敗することがあります。

MP3やWAVに変換してから試すと安定する場合があります。

### 動きプレビューが静かすぎる

`動きプレビューに適した区間を自動検出` をオンにしてください。  
静かなイントロではなく、音が大きい区間を自動的に使います。

### 30秒プレビューが静かなイントロだけになる

これは仕様です。  
30秒プレビューMP4は、動画編集ソフトのタイムライン先頭で試すため、常に音源の先頭から生成します。

GUI内の動きプレビューとは目的が違います。

### 無音部分でバーが残る

v1.0.0では、無音・ほぼ無音ではバーが消えるように調整しています。  
それでも残る場合は、音源に実際のノイズや残響が含まれている可能性があります。

詳細設定の `min dB` や `Base cut` を調整すると改善することがあります。

### バーが低すぎる

生成・動きタブの `感度` を `高め` にしてください。  
または詳細設定の `gain_db` を上げてください。

### バーが高すぎる

生成・動きタブの `感度` を `低め` にしてください。  
または詳細設定の `gain_db` を下げてください。

### バーの動きが鈍い

生成・動きタブの `反応の速さ` を `速い` にしてください。  
または `動きの細かさ` を `細かい` にしてください。

### バーが暴れすぎる

`反応の速さ` を `ゆったり` にするか、`03 Dynamic Smooth` を使ってください。

### h264_nvenc で失敗する

GPUやffmpegビルドがNVENCに対応していない可能性があります。  
標準の `libx264` に戻してください。

### 生成ファイルが大きい

CRFを上げるとファイルサイズが小さくなります。  
ただし、上げすぎると画質が落ちます。

標準は18です。  
20〜23程度でも、この用途では問題ない場合があります。

### 動画編集ソフトで黒背景が残る

合成モードを確認してください。  
通常は、比較（明）/ Lighten / Screen 系を使います。

クロマキーで黒を抜く方法もありますが、白バーのエッジが不自然になる場合があります。

### 動画編集ソフトでシンクロしない

次を確認してください。

- 元音源とスペアナMP4を同じ開始位置に置いているか。
- スペアナMP4を途中で伸縮していないか。
- 音源を別途編集して長さが変わっていないか。
- fps変換やタイムライン設定でズレが出ていないか。

Audio Spectrum Overlay Maker の全体生成は、音源長に対応するフレームを生成します。動画編集ソフト側で時間伸縮しなければ、基本的には最後までシンクロします。

## 内部処理の考え方

v1.0.0では、内部処理を次のように分けています。

```text
解析層
↓
変形層
↓
描画層
```

### 解析層

音源から、内部解析値 `SpectrumData` を作ります。

`SpectrumData` には次の情報が含まれます。

- Dynamic処理後の内部値
- Dynamic処理前のdB値
- 周波数境界
- fps
- sample rate
- internal analysis bands
- frequency range

### 変形層

内部解析値を、実際に表示するバー本数へ変換します。

表示バー数が18でも64でも、内部解析の粒度を分離しているため、動きの印象を近づけやすくなっています。

### 描画層

変形済みの値を、実際のフレーム画像に描画します。

v1.0.0では、横型の白い角丸バーを描画します。  
角丸バーは、小さい音量でいきなりフル幅の四角として出ないよう、中心から小さなレモン/ピル形状で出現し、高さが上がるにつれて通常幅へ広がります。

### Dynamic処理

Dynamic処理は、単純な音量表示ではなく、映像素材として気持ちよく見えるように調整した処理です。

主な特徴：

- 各フレーム内の相対ピークを基準にする。
- 持続音をそのまま張り付かせず、新しいピークを強調する。
- Attack / Release で上下の追従を調整する。
- 無音・ほぼ無音では絶対音量ゲートによりバーを消す。
